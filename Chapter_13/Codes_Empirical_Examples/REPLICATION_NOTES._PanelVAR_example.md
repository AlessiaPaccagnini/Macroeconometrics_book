## 1. Which language is canonical?

**Python is the baseline.** All numbers and figures printed in the textbook
(Table 13.6 and Figure 13.1) are generated by the Python script. The R and
MATLAB scripts are faithful **ports of the same methodology**: they implement
identical models, estimators, identification, and inference, and they
reproduce the same **qualitative** findings. They do **not** reproduce the
exact decimal values in the book, and they are not expected to. This document
explains why.

---

## 2. Why don't the three languages give identical numbers?

The empirical examples use **simulated** (calibrated) data. The data are
generated inside each script by drawing pseudo-random numbers from a fixed
seed (`seed = 42`). The catch is that **each language uses a different
pseudo-random number generator (PRNG)**:

| Language | Default generator | Seeding call |
|----------|-------------------|--------------|
| Python (NumPy) | PCG64 (`default_rng`) | `np.random.default_rng(42)` |
| R | Mersenne-Twister | `set.seed(42)` |
| MATLAB | Mersenne-Twister (MATLAB variant) | `rng(42)` |

Even though every script requests the *same* random draws in the *same order*
(the oil-price path, the country fixed effects, and each country's shocks),
the underlying generators return *different numbers* for the same seed.
Because the entire simulated panel is built from those draws, the three
languages simulate **three different datasets**. Different data produce
different coefficient estimates, t-statistics, Wald statistics, impulse
responses, and bootstrap bands.

This is not a bug. It is an inherent property of cross-language replication
with internally simulated data. The same script run on the same machine in
the same language *will* reproduce its own results exactly; only
*cross-language* agreement is impossible without sharing the data (see §5).

## 3. What agrees and what differs

**Agrees across all three languages (qualitative findings):**

- Oil-importing economies show a **negative** GDP response to a positive oil
  price shock; oil **exporters** show a **positive** response.
- The sign asymmetry between groups is the central result, and the
  interacted Panel VAR recovers it in every language.
- The **pooled** specification masks this heterogeneity: the pooled GDP
  response is small and near-flat, sitting between the two groups.
- Inflation rises in both groups, with importers peaking slightly higher and
  earlier.
- The Wald test for coefficient homogeneity (H0: D1 = D2 = 0) is **strongly
  rejected** in every language (p < 0.001).
- The interest-rate persistence and the small-exporter-group imprecision
  (N_exp = 4) appear in all three.

**Differs across languages (exact magnitudes):**

The table below shows the GDP-equation oil coefficient (first lag) and the
Wald statistic actually produced by each script, alongside the published
(Python) values. Magnitudes are of the same order and sign; decimals differ.

| Quantity | Python (book) | R | MATLAB |
|----------|--------------:|------:|-------:|
| Pooled Δp^oil → GDP | −0.012 (t=−2.03) | −0.009 (t=−1.58) | −0.004 (t=−0.64) |
| Importers | −0.024 (t=−6.21) | −0.021 (t=−6.53) | −0.018 (t=−4.78) |
| Exporters | +0.014 (t=+1.79) | +0.017 (t=+4.38) | +0.026 (t=+3.29) |
| Wald χ²(18) | 192.4 | 159.2 | 80.5 |
| Importer GDP trough | ≈ −0.002 pp (h=2) | ≈ −0.002 pp (h=2) | ≈ −0.002 pp (h=2) |

(MATLAB values are from the desktop MATLAB run; note that Octave — an
open-source MATLAB-compatible interpreter — uses yet another PRNG and will
produce different numbers again.)

## 4. Inference: the bootstrap

All three scripts compute 90% pointwise confidence bands for the impulse
responses using a **sieve bootstrap on the estimated residuals**, consistent
with the procedure described in Appendix 13A (Dees et al. 2007). The sieve
bootstrap fits a short auxiliary VAR to the residuals, resamples the whitened
innovations, and **regenerates each country's series through the estimated
dynamics**, thereby preserving serial dependence.

Two implementation details to be aware of:

1. **The plotted line is the point estimate**, not a bias-corrected line. The
   bootstrap is used only to construct the bands. This keeps the plotted
   trough/peak consistent with the estimated coefficients.
2. **Bands are recentred (pivotal).** With a very small group (the exporter
   group has only N = 4 units), raw percentile bands can drift away from — or
   even off the sign of — the point estimate. Centring the bootstrap
   dispersion on the point estimate yields bands that bracket the estimate
   while still reflecting sampling uncertainty. This is most visible in the
   exporter panels and is itself a useful illustration of the small-group
   imprecision discussed in the chapter.

Because the bootstrap also draws random numbers, band *widths* differ across
languages for the same reason coefficients do.

## 5. If you need exact cross-language agreement

For most teaching and replication purposes, qualitative agreement is the
right standard and no further action is needed. If exact agreement is
required (e.g. for a referee), the only reliable route is to **share the
data** rather than the seed:

1. Run the Python script once and export the simulated panel to CSV
   (one file, all 13 countries, columns: country, date, gdp, inf, oil).
2. Modify the R and MATLAB scripts to **load that CSV** instead of calling
   their own `simulate_panel`.
3. All three then estimate on identical data and reproduce the published
   Table 13.6 and Figure 13.1 to the printed precision.

The trade-off is pedagogical: students no longer see the data-generating
process written in their own language. For that reason the default companion
scripts simulate internally, and this document explains the resulting
differences.
